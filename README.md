# Log Error Analyzer

A small Python CLI tool that sends a log file to a local [Ollama](https://ollama.com/) model and asks it to analyze the error, summarize the root cause, and suggest fixes.

## How it works

The script reads the content of the provided log file, builds a prompt for the model, and uses the Ollama Python SDK to get a structured analysis with the following sections:

- Root Cause
- Likely Failing File or Function
- Immediate Fix
- Commands to Try
- Prevention
- Severity

## Requirements

- Python 3.8+
- [Ollama](https://ollama.com/) installed and running locally
- The model `gemma4:e4b` pulled in Ollama (or change the model name in the script)

## Installation

1. Clone the repository:

   ```bash
   git clone https://github.com/yourusername/log-error-analyzer.git
   cd log-error-analyzer
   ```

2. Install the Python dependencies:

   ```bash
   pip install ollama
   ```

3. Make sure Ollama is running and the model is available:

   ```bash
   ollama pull gemma4:e4b
   ```

## Usage

```bash
python log_error.py <log_file>
```

### Example

```bash
python log_error.py log.txt
```

The script will print the prompt and the model's analysis to the console.

## Configuration

The model name is defined in the `MODEL_NAME` variable at the top of `log_error.py`. You can change it to any model available in your Ollama environment.

```python
MODEL_NAME = "gemma4:e4b"
```

## Files

- `log_error.py` — main script
- `log.txt` — example log file
- `README.md` — this file

## Notes

- The analysis is generated by a local LLM. Always verify the model's suggestions before applying them.
- The script does not send data to external APIs; everything is processed by your local Ollama instance.
